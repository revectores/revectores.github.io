1. Overview

1. 概述

1. 概要

poi-manager is a personal place-tracking tool built around a map interface. The idea is simple: keep a private library of places you've been — or want to go — searchable and browsable on a full-screen map, with per-place visit logs (date + notes) and visual distinction between visited and not-yet-visited markers.

poi-manager 是一个以地图为核心入口的个人地点记录工具。 设计思路简单：维护一个私有的地点库，记录你去过或想去的地方，支持在全屏地图上搜索和浏览， 为每个地点记录多次到访（时间 + 备注），并用视觉区分"已去过"和"未去过"的标记。

poi-manager は地図インターフェースを中心に据えた個人スポット管理ツールです。 コンセプトはシンプルです。訪れた場所や行きたい場所のプライベートライブラリを持ち、 フルスクリーンの地図で検索・ブラウズでき、スポットごとに複数回の訪問ログ（日時＋メモ）を記録し、 訪問済みと未訪問のマーカーを視覚的に区別します。

2. Features

2. 功能

2. 機能

  • Map main view — full-screen Amap (高德地图) showing all imported places as markers.
  • POI search — search any location via the Amap API; results appear as pins on the map before importing.
  • One-click import — add a search result to your personal library with a single action.
  • Visit log — record multiple visits per place, each with a timestamp and optional notes.
  • Status filter — toggle the map to show only visited, only unvisited, or all places.
  • Visual distinction — visited places shown in blue, unvisited in red.
  • 地图主界面 — 全屏高德地图，以标记点展示所有已导入的地点。
  • POI 搜索 — 通过高德地图 API 搜索任意地点，搜索结果在导入前以图钉形式显示在地图上。
  • 一键导入 — 单次操作即可将搜索结果添加到个人库。
  • 到访记录 — 为每个地点记录多次到访，每次含时间戳和可选备注。
  • 状态筛选 — 切换地图显示：仅已去过、仅未去过，或全部显示。
  • 视觉区分 — 已去过的地点显示为蓝色，未去过的显示为红色。
  • 地図メイン画面 — フルスクリーンの高德地図（Amap）にインポートしたすべてのスポットをマーカー表示。
  • POI 検索 — Amap API で任意のスポットを検索し、インポート前に地図上にピンとして表示。
  • ワンクリックインポート — 検索結果を1操作でパーソナルライブラリに追加。
  • 訪問ログ — スポットごとに複数回の訪問を記録。各訪問にタイムスタンプと任意メモを付与。
  • ステータスフィルタ — 訪問済みのみ・未訪問のみ・全件表示を地図上で切り替え。
  • 視覚的区別 — 訪問済みは青、未訪問は赤で表示。

4. Getting Started

4. 快速开始

4. はじめ方

You need an Amap JS API key from the Amap developer console. Then:

你需要在高德开放平台申请 JS API Key，然后：

Amap デベロッパーコンソールから JS API キーを取得してから：

cp .env.local.example .env.local
# fill in NEXT_PUBLIC_AMAP_KEY and NEXT_PUBLIC_AMAP_SECURITY_CODE

npm install
npm run db:push   # sync schema to SQLite
npm run dev       # http://localhost:3000

To inspect the database visually:

可视化查看数据库：

データベースをビジュアルで確認するには：

npm run db:studio
